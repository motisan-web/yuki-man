# ゆうきまん 記事追加モード

> **このファイルだけ読めば記事を追加できます。CLAUDE.md・CURRENT.mdは読まなくて構いません。**

---

## ゆうきまんとは

日本版フロリダマン的な概念のニュースサイト。日常のちょっとしたすれ違いや善意がズレた方向に転がる、実害は少ないが妙に納得感のある事件を扱う。

---

## 記事のトーン・スタイル

- 実在しない人物・事件（フィクション）。記事末尾に免責文を入れる
- タイトルは「状況 → 結末 + 当事者属性（年齢）」形式が基本
  - 例：`自販機に「500円返せ」→3時間交渉、缶コーヒー9本を得て「勝った」と主張する男（37）`
- 悪意のある犯罪・暴力系ではなく「善意・不注意・思い込みがズレた」系が主軸
- 本文は報道記事風。「〜という。」「〜と供述している。」「〜とコメントした。」
- 見出しは `## 事件の概要` `## 経緯` `## その後の展開` などで分ける
- 分量：600〜1200字程度（見出し3〜4個）
- 末尾に必ず `---\n\n*この事件は実在の人物・団体とは関係ありません。*` を入れる

### タグの方針（2〜4個）
事件の性質を表す日本語タグ。例：`逮捕` `善意` `うっかり` `食べ物` `警察` `裁判` `勘違い` `近所づきあい` など

---

## 手順

### Step 1 — MDファイルを作成

パス：`articles/YYYY/MM/DD/slug.md`

- `YYYY/MM/DD` は記事の日付
- `slug` はケバブケース英数字（記事内容を表す英語）
  - 例：`woman-burns-gyoza-after-groping`

**フロントマターテンプレート：**
```
---
title: タイトル
date: YYYY-MM-DD
slug: kebab-case-slug
tags: [タグ1, タグ2]
thumbnail: 
summary: 要約文（80字以内、検索・一覧表示に使用）
published: true
---
本文（Markdown）
```

---

### Step 2 — data/index.json の先頭にエントリを追加

**追加するエントリの形式：**
```json
{
    "slug": "kebab-case-slug",
    "title": "タイトル",
    "date": "YYYY-MM-DD",
    "tags": ["タグ1", "タグ2"],
    "thumbnail": "",
    "summary": "要約文",
    "path": "articles/YYYY/MM/DD/kebab-case-slug.md",
    "published": true
}
```

- 配列の**先頭**に追加する（date降順を維持）
- 同じ日付が複数ある場合も先頭でOK

---

### Step 3 — git コミット & プッシュ

```
git add articles/YYYY/MM/DD/slug.md data/index.json
git commit -m "content: 記事追加「タイトルの短縮形」"
git push origin main
```
