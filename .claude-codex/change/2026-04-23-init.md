# プロジェクト初期化 2026-04-23

## 概要
yuki-man プロジェクトの設計ドキュメント作成。実装はまだ行っていない。

---

## 決定した仕様

### コンセプト
- 「ゆきまん」= 日本版フロリダマン。日常のちょっとした事件がズレた方向に転がる話。
- 例: 痴漢被害にあった女性が怒りで注意散漫になり餃子を焦がして火事 → 逮捕
- Zenn + 日本ニュースサイト的なシンプル清潔感のあるデザイン

### アーキテクチャ判断
- **フレームワークなし**: Xserverの制約を考慮し、Composer不要の素PHP構成
- **Parsedown**: Markdownパーサーとして単一ファイル版を使用（vendor不要）
- **JSONインデックス**: DBなしで全文検索を実現するため `data/index.json` を維持
- **管理認証**: セッションベースの簡易認証（`admin/` 以下を保護）
- **画像**: `thumbnails/` に手動アップロード or admin経由アップロード

### 記事ディレクトリ設計
```
articles/
  2026/
    04/
      23/
        woman-burns-gyoza-after-groping-incident.md
```
日付をパスに含めることで、日付ブラウズ（/YYYY/MM/DD/）が容易になる。

### 検索設計
- `data/index.json` にtitle・summary・tagsをインデックス化
- PHP側でjson_decode後にarray_filterで部分一致検索
- 将来的にSQLiteに移行できる設計にしておく（interfaceを切る）

### 管理画面設計
- `/admin/` → ログイン後、記事一覧
- `/admin/edit.php?slug=xxx` → 編集（新規はslugなし）
- MDエディタ: textarea + プレビュー（marked.js）
- 画像アップロード: フォームからthumbnails/へ

### セキュリティ考慮
- admin/はセッション認証必須
- ファイルアップロードはMIMEチェック + 拡張子制限（jpg/png/gif/webp）
- MDの保存はファイル名をスラッグ正規化（英数字ハイフンのみ）
